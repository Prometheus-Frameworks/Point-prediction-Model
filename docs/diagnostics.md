# Projection diagnostics

## Purpose
PR12 adds a deterministic diagnostics layer for WR/TE outputs that sits on top of the existing projection and interval stack. The diagnostics layer does **not** train a second model. Instead, it interprets:
- the point projection,
- residual-based prediction intervals,
- recent usage trends,
- recent efficiency trends,
- event context,
- sample reliability and volatility,

to produce structured regression-up, regression-down, stickiness, and fragility outputs.

## Output shape
Each diagnostic record includes:
- `regressionUpScore`
- `regressionDownScore`
- `stickinessScore`
- `fragilityScore`
- component-level scorer outputs
- machine-readable flags
- explanation bullets
- interval width summaries

## Scoring philosophy
The layer is intentionally rule-based and interpretable.

### 1. Usage vs production gap
This component compares recent role/volume indicators such as targets, routes, target share, and TPRR against recent fantasy production and projection level.
- High usage with weaker recent fantasy conversion pushes toward **regression up**.
- Lower usage with unusually strong recent fantasy output pushes toward **regression down**.

### 2. Efficiency fragility
This component checks whether recent efficiency is running ahead of the role that supports it.
- Strong YPRR/YPT/catch-rate growth with weaker target support, lower reliability, or higher volatility increases fragility.
- When role support is stronger than recent efficiency, the signal is less fragile and can lean back toward upside.

### 3. Touchdown regression risk
This component separates touchdown downside and upside:
- `downScore` rises when TD-per-target and TD dependence are high relative to the total projection.
- `upScore` rises when red-zone and end-zone usage are present but recent TD conversion is lagging.

### 4. Volume stability
This component focuses on the underlying role:
- Stable routes, target share, snap rate, and game participation increase stability.
- Usage volatility, role swing, and thin sample support increase fragility.

### 5. Projection stickiness / fragility
This component interprets the projection interval and contextual instability:
- Narrower intervals, stronger reliability, and calmer context improve stickiness.
- Wide intervals, recent event turbulence, team changes, and noisy usage increase fragility.

## Flag generation
Flags are threshold-based and machine-readable so downstream systems can present them directly.

Examples:
- `REGRESSION_UP_USAGE_STRONG`
- `REGRESSION_DOWN_TD_FRAGILE`
- `PROJECTION_STICKY_HIGH_VOLUME`
- `PROJECTION_FRAGILE_EVENT_DRIVEN`
- `EFFICIENCY_AHEAD_OF_ROLE`
- `USAGE_AHEAD_OF_PRODUCTION`

Flags are generated by combining component thresholds with contextual checks. For example:
- strong positive usage gaps trigger usage-led upside flags,
- elevated TD dependence plus fragility triggers downside TD flags,
- high stickiness plus stable volume triggers sticky-projection flags,
- recent non-`NONE` events plus high fragility trigger event-driven fragility flags.

## Service entrypoints
Two service-level entrypoints are available:

### `scoreRegressionCandidatesService`
Use this when you already have model outputs. It accepts `(row, prediction)` pairs and scores them directly.

### `runProjectionDiagnosticsService`
Use this when you want the diagnostics layer to call the existing model inference flow first. It:
1. runs interval-aware predictions through `predictWithIntervalsService`,
2. scores each resulting output deterministically,
3. returns a single service envelope with rows, predictions, and diagnostics.

## Examples included
`src/diagnostics/examples/sampleDiagnosticsRun.ts` includes realistic examples for:

- underproducing high-usage player
- overproducing low-usage player
- event-driven high-uncertainty player

## Limitations
These diagnostics are heuristics, not probabilistic truth.
- They do not observe live market sentiment or consensus ranks.
- They do not ingest new external data sources in this PR.
- They can identify interpretable tension between role and output, but they cannot prove a player will regress in a given week.
- They inherit whatever quality and coverage exist in the current feature pipeline and interval artifact.
